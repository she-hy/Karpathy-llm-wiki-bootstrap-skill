# BM25 Search Layer

This directory contains the local BM25 full-text search index for the wiki.

BM25 is optional. It helps the LLM find relevant Markdown passages once the
wiki grows beyond what `wiki/index.md` and `rg` can comfortably cover.

## What BM25 Adds

- Fast local keyword search over wiki pages
- Better recall for names, products, terms, paper titles, APIs, and distinctive wording
- A rebuildable navigation layer for query, ingest, and lint workflows
- Exportable chunks for audit, migration, or debugging

## What BM25 Does Not Do

- It does not replace `wiki/` as the knowledge layer.
- It does not replace `SCHEMA.md` as the operating contract.
- It does not judge whether a claim is true.
- It does not understand synonyms as well as embeddings.
- It does not produce final answers. The LLM must read the returned wiki pages
  and synthesize an answer with normal wiki citations.

## LLM and BM25 Cooperation

```text
User question
  -> LLM extracts terms, entities, and intent
  -> BM25 returns candidate chunks
  -> LLM opens the matching wiki pages
  -> LLM reads full context and follows wikilinks
  -> LLM answers with citations to wiki pages
```

BM25 is a retrieval helper. The wiki remains the source of truth.

## Normal Query vs Export Use

**Normal Query** is how the LLM answers user questions.

1. Read `wiki/index.md` first.
2. Run `python3 scripts/wiki_fts.py stats`.
3. If `Fresh: no`, run `python3 scripts/wiki_fts.py rebuild`; if rebuild fails and fallback is allowed, use `wiki/index.md` plus `rg`.
4. Run `python3 scripts/wiki_fts.py search "{query}" --limit 10`.
5. Treat returned chunks as candidate locations only.
6. Open the returned `page_path` files under `wiki/` and read the relevant full context.
7. Follow useful wikilinks from those pages.
8. Answer with citations to wiki pages, not to BM25 rows.

**Export Use** is an operations and audit mechanism.

- Use exports for audit, migration, debugging, external analysis, or inspecting what text was indexed.
- Do not use exports as the normal query path when `search` is available.
- Do not cite `exports/*`, `indexes/fts.sqlite`, `chunk_id`, or BM25 `score` in user-facing answers.
- Treat exports as sensitive because they contain indexed wiki text.

## Search Output Contract

`search` prints ranked candidate chunks:

```text
1. {title} [{page_path}]
   heading: {heading_path}
   chunk: {chunk_id} score: {score}
   {snippet}
```

| Field | Meaning | How the LLM should use it |
| --- | --- | --- |
| `title` | Page title from frontmatter or first heading. | Human-readable label only. |
| `page_path` | Relative path from the generated wiki/project root, including the `wiki/` prefix, such as `wiki/concepts/rag.md`. | Open and read this file before answering. |
| `heading_path` | Heading trail inside the page, such as `Overview > Limitations`. | Helps locate the relevant section after opening the page. |
| `chunk_id` | Stable chunk identifier: `{page_path}#{ordinal}`. | Diagnostic handle only; do not cite it as evidence. |
| `score` | SQLite FTS5 BM25 score for this query. Lower is more relevant; values may be negative. | Rank candidates within this query only. Do not compare across unrelated queries. |
| `snippet` | Text from the indexed chunk, truncated for display. | Preview only. Never answer from snippet alone. |

Current search tokenizes the query into terms and combines them with OR. For
precision, run several focused searches using exact page titles, entity names,
or distinctive phrases from `wiki/index.md`.

## Data Model

The SQLite database is `indexes/fts.sqlite`. It is rebuildable from `wiki/` and
is generated by `scripts/wiki_fts.py`. The current schema version is `1`.

### Table: `pages`

One row per Markdown page under `wiki/`.

| Field | Type | Definition |
| --- | --- | --- |
| `page_path` | `TEXT PRIMARY KEY` | Relative path from the generated wiki/project root, including the `wiki/` prefix, such as `wiki/concepts/rag.md`. |
| `title` | `TEXT NOT NULL` | Page title from frontmatter `title`, first `#` heading, or filename fallback. |
| `type` | `TEXT NOT NULL` | Page type from frontmatter `type`, such as `concept`, `entity`, or `source-summary`. |
| `updated` | `TEXT NOT NULL` | Frontmatter `updated` value. Empty string if absent. |
| `sources` | `TEXT NOT NULL` | Raw frontmatter `sources` value, preserved as text. |
| `tags` | `TEXT NOT NULL` | Raw frontmatter `tags` value, preserved as text. |
| `mtime` | `REAL NOT NULL` | Filesystem modification time used for freshness checks. |

### Table: `chunks`

One row per indexed text chunk. Chunks are split from each page by heading
section and character window.

| Field | Type | Definition |
| --- | --- | --- |
| `chunk_id` | `TEXT PRIMARY KEY` | Stable identifier `{page_path}#{ordinal}`. |
| `page_path` | `TEXT NOT NULL` | Parent page path; matches `pages.page_path`. |
| `title` | `TEXT NOT NULL` | Parent page title copied onto each chunk. |
| `type` | `TEXT NOT NULL` | Parent page type copied onto each chunk. |
| `heading_path` | `TEXT NOT NULL` | Heading trail for the chunk's section. |
| `ordinal` | `INTEGER NOT NULL` | Chunk number within the parent page, starting at 1. |
| `sources` | `TEXT NOT NULL` | Parent page `sources` copied onto each chunk. |
| `tags` | `TEXT NOT NULL` | Parent page `tags` copied onto each chunk. |
| `updated` | `TEXT NOT NULL` | Parent page `updated` copied onto each chunk. |
| `text` | `TEXT NOT NULL` | Markdown-stripped chunk text used for search and export. |

### Virtual Table: `chunks_fts`

SQLite FTS5 table used for ranking chunks.

| Field | Indexed | Definition |
| --- | --- | --- |
| `chunk_id` | No (`UNINDEXED`) | Diagnostic join key copied from `chunks.chunk_id`. |
| `title` | Yes | Page title searchable text. |
| `page_path` | No (`UNINDEXED`) | Page path returned for navigation. |
| `heading_path` | Yes | Heading trail searchable text. |
| `text` | Yes | Main searchable chunk text. |

Search joins `chunks_fts` back to `chunks` by `chunk_id` so the LLM can see
metadata and preview text. The BM25 `score` is computed at query time from
`chunks_fts`; it is not stored in `chunks` or exported.

### Table: `metadata`

Key-value metadata about the current build.

| Key | Meaning |
| --- | --- |
| `built_at` | UTC timestamp when the index was built. |
| `schema_version` | Data model version; currently `1`. |
| `root` | Absolute wiki root used during build. |
| `page_count` | Number of indexed wiki pages. |
| `chunk_count` | Number of indexed chunks. |

### Export Fields

JSONL, CSV, and Markdown exports contain rows from `chunks` in this order:

```text
chunk_id, page_path, title, type, heading_path, ordinal, sources, tags, updated, text
```

Exports do not include `score`, `mtime`, or FTS internals. Scores are
query-local and are computed only by `search`.

## Commands

Run commands from the wiki root:

```bash
python3 scripts/wiki_fts.py doctor
python3 scripts/wiki_fts.py build
python3 scripts/wiki_fts.py rebuild
python3 scripts/wiki_fts.py search "query text" --limit 10
python3 scripts/wiki_fts.py stats
```

Export the full indexed corpus:

```bash
python3 scripts/wiki_fts.py export --format jsonl --out exports/bm25-chunks.jsonl
python3 scripts/wiki_fts.py export --format csv --out exports/bm25-chunks.csv
python3 scripts/wiki_fts.py export --format markdown --out exports/bm25-report.md
```

Exports contain chunk rows with page metadata fields and chunk text. BM25
scores are not exported because BM25 scores are computed at query time.

## Git Policy

Commit:

- `scripts/wiki_fts.py`
- `indexes/README.md`
- `.llm-wiki-bootstrap/EXTEND.md` if the preferences are project-specific

Usually do not commit:

- `indexes/fts.sqlite`
- `exports/*.jsonl`
- `exports/*.csv`
- `exports/*.md`

The database and exports are rebuildable from `wiki/`.

## Privacy

`fts.sqlite` and exported files contain searchable text from the wiki. If the
wiki contains private notes, customer material, personal journals, or sensitive
research, do not share these files casually.

## Troubleshooting

If `doctor` fails, the local SQLite build does not include FTS5. Use a Python
distribution with SQLite FTS5 enabled, or continue with `index.md + rg`.

If `stats` says `Fresh: no`, rebuild:

```bash
python3 scripts/wiki_fts.py rebuild
```

If search returns nothing, try exact terms from the wiki, product names, paper
titles, or run `rg` as a fallback.
