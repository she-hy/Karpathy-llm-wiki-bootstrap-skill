# BM25 Workflow

BM25 is an optional local full-text search layer for larger LLM Wikis. It is a
navigation layer, not a source of truth.

## Preference Loading

Before bootstrap, ingest, query, lint, or BM25 work, read `EXTEND.md` using the
priority order in `references/config/extend-schema.md`.

If no `EXTEND.md` exists, run first-time preference setup and create one from
`references/templates/extend.md`. Do not silently use defaults.

## Trigger Check

When `bm25.mode: auto_prompt`, compute:

- number of files under `raw/` excluding `.gitkeep`
- number of Markdown files under `wiki/`
- total characters across `wiki/**/*.md`
- line count of `wiki/index.md`
- pages read in the most recent query, when known

If any normal threshold in `bm25.thresholds` is reached, ask whether to
initialize BM25. If `prompt_once: true` and the user already declined at normal
thresholds, do not repeat until a strong threshold is reached.

If any strong threshold is reached, ask again even after a previous decline.

If `bm25.mode` is:

- `manual`: do not prompt; initialize only on explicit request.
- `off`: do not check or prompt.
- `enabled`: use BM25 if present; ask to initialize if missing.
- `required`: require BM25 for ingest/query/lint and stop if unavailable.

## Initialization

Only initialize after explicit user approval.

Create or update:

```text
scripts/wiki_fts.py
indexes/
indexes/README.md
exports/
.gitignore
SCHEMA.md
wiki/log.md
```

Use templates:

- `references/templates/wiki_fts.py` -> `scripts/wiki_fts.py`
- `references/templates/bm25-readme.md` -> `indexes/README.md`

`references/templates/wiki_fts.py` stays under templates because it is copied
into the generated wiki. It is not a skill-owned helper executed from the
installed skill package.

Append these ignore rules if missing:

```gitignore
indexes/fts.sqlite
exports/*.jsonl
exports/*.csv
exports/*.md
```

Then run:

```bash
python3 scripts/wiki_fts.py doctor
python3 scripts/wiki_fts.py build
python3 scripts/wiki_fts.py stats
python3 scripts/wiki_fts.py search "LLM Wiki" --limit 5
python3 scripts/wiki_fts.py export --format jsonl --out exports/bm25-chunks.jsonl
```

If the smoke-test query term is not present in the wiki, choose a title or
concept from `wiki/index.md`.

## Schema Update

Add a `Search Layer` section to `SCHEMA.md`:

````markdown
## Search Layer

BM25 is optional and local. It helps the LLM find candidate wiki passages.
It is not a source of truth and does not replace reading wiki pages.

When BM25 is enabled:
- Query: read `wiki/index.md`, run `python3 scripts/wiki_fts.py stats`, rebuild if `Fresh: no`, run `python3 scripts/wiki_fts.py search "{query}" --limit 10`, open returned pages, follow relevant wikilinks, then answer with citations to wiki pages.
- Ingest: before creating new entity, concept, claim, method, or synthesis pages, search BM25 for existing related pages to avoid duplicates.
- Lint: run `python3 scripts/wiki_fts.py stats`; if stale, rebuild or record a warning.
- Fallback: if BM25 fails and preferences allow fallback, use `rg` and `wiki/index.md`.

Normal Query means LLM question answering: check freshness with `stats`, rebuild stale indexes when possible, use `search`, open returned `page_path` files, read full wiki context, and cite wiki pages. Export Use means operations, audit, migration, debugging, or external analysis; do not use exports as the normal query path when `search` is available.

Search output fields:

| Field | Meaning | Use |
| --- | --- | --- |
| `page_path` | Wiki page containing the match. | Open this page before answering. |
| `heading_path` | Heading trail within the page. | Locate the relevant section. |
| `chunk_id` | `{page_path}#{ordinal}` diagnostic identifier. | Do not cite. |
| `score` | Query-local SQLite FTS5 BM25 score; lower is more relevant and values may be negative. | Rank candidates within one query only. |
| `snippet` | Truncated chunk preview. | Never answer from snippet alone. |

Never cite `chunk_id`, `score`, `indexes/fts.sqlite`, or `exports/*` as evidence in user-facing answers. BM25 data finds candidate pages; wiki pages are the citation targets.

The BM25 data model is documented in `indexes/README.md`. Its exported rows have exactly these fields:

```text
chunk_id, page_path, title, type, heading_path, ordinal, sources, tags, updated, text
```
````

## Query Use

1. Read `wiki/index.md` for structure.
2. Extract query terms, entities, and likely page names.
3. Run `python3 scripts/wiki_fts.py stats`.
4. If `Fresh: no`, run `python3 scripts/wiki_fts.py rebuild`; if rebuild fails and fallback is allowed, use `rg` and index navigation.
5. Run BM25 search for the user question and key terms.
6. Open the returned wiki pages; do not answer from snippets alone.
7. Follow relevant wikilinks.
8. Answer with citations to wiki pages.
9. If BM25 has no useful result, fallback to `rg` and index navigation.

## Normal Query vs Export Use

**Normal Query** means LLM question answering. Use `stats` to check freshness,
rebuild stale indexes when possible, use `search`, open returned `page_path`
files, read full wiki context, then cite wiki pages.

**Export Use** means operations, audit, migration, debugging, or external
analysis. Exports are not the normal query path when `search` is available.

Do not cite BM25 internals in user-facing answers: no `chunk_id`, no `score`,
no `fts.sqlite`, and no `exports/*` as evidence. BM25 data locates candidate
wiki pages; the wiki pages are the citation targets.

## Search Output Contract

`search` prints ranked candidate chunks:

```text
1. {title} [{page_path}]
   heading: {heading_path}
   chunk: {chunk_id} score: {score}
   {snippet}
```

| Field | Meaning | LLM usage |
| --- | --- | --- |
| `title` | Page title from frontmatter or first heading. | Label only. |
| `page_path` | Relative path from the generated wiki/project root, including the `wiki/` prefix. | Open this page before answering. |
| `heading_path` | Heading trail inside the page. | Locate relevant section after opening the page. |
| `chunk_id` | `{page_path}#{ordinal}`. | Diagnostic handle only; do not cite. |
| `score` | SQLite FTS5 BM25 score. Lower is more relevant; values may be negative. | Rank candidates within one query only. |
| `snippet` | Truncated chunk text. | Preview only; never answer from it alone. |

Current search tokenizes the query into terms and combines them with OR. For
precision, run several focused searches using exact page titles, entity names,
or distinctive phrases from `wiki/index.md`.

## Data Model Contract

The generated SQLite database is `indexes/fts.sqlite`, built by
`scripts/wiki_fts.py` from Markdown pages under `wiki/`. Current schema version:
`1`.

### `pages`

One row per Markdown page under `wiki/`.

| Field | Type | Definition |
| --- | --- | --- |
| `page_path` | `TEXT PRIMARY KEY` | Relative path from the generated wiki/project root, including the `wiki/` prefix, e.g. `wiki/concepts/rag.md`. |
| `title` | `TEXT NOT NULL` | Frontmatter `title`, first `#` heading, or filename fallback. |
| `type` | `TEXT NOT NULL` | Frontmatter `type`. |
| `updated` | `TEXT NOT NULL` | Frontmatter `updated`, or empty string. |
| `sources` | `TEXT NOT NULL` | Raw frontmatter `sources` preserved as text. |
| `tags` | `TEXT NOT NULL` | Raw frontmatter `tags` preserved as text. |
| `mtime` | `REAL NOT NULL` | Filesystem modification time used by freshness checks. |

### `chunks`

One row per indexed text chunk.

| Field | Type | Definition |
| --- | --- | --- |
| `chunk_id` | `TEXT PRIMARY KEY` | Stable identifier `{page_path}#{ordinal}`. |
| `page_path` | `TEXT NOT NULL` | Parent page path; matches `pages.page_path`. |
| `title` | `TEXT NOT NULL` | Parent page title copied onto the chunk. |
| `type` | `TEXT NOT NULL` | Parent page type copied onto the chunk. |
| `heading_path` | `TEXT NOT NULL` | Heading trail for the chunk section. |
| `ordinal` | `INTEGER NOT NULL` | Chunk number within the parent page, starting at 1. |
| `sources` | `TEXT NOT NULL` | Parent page `sources` copied onto the chunk. |
| `tags` | `TEXT NOT NULL` | Parent page `tags` copied onto the chunk. |
| `updated` | `TEXT NOT NULL` | Parent page `updated` copied onto the chunk. |
| `text` | `TEXT NOT NULL` | Markdown-stripped chunk text used for search and export. |

### `chunks_fts`

SQLite FTS5 virtual table used for ranking chunks.

| Field | Indexed | Definition |
| --- | --- | --- |
| `chunk_id` | No (`UNINDEXED`) | Diagnostic join key copied from `chunks.chunk_id`. |
| `title` | Yes | Page title searchable text. |
| `page_path` | No (`UNINDEXED`) | Page path returned for navigation. |
| `heading_path` | Yes | Heading trail searchable text. |
| `text` | Yes | Main searchable chunk text. |

Search joins `chunks_fts` to `chunks` by `chunk_id`. The BM25 `score` is
computed at query time from `chunks_fts`; it is not stored and is not exported.

### `metadata`

| Key | Meaning |
| --- | --- |
| `built_at` | UTC timestamp when the index was built. |
| `schema_version` | Data model version; currently `1`. |
| `root` | Absolute wiki root used during build. |
| `page_count` | Number of indexed wiki pages. |
| `chunk_count` | Number of indexed chunks. |

### Export fields

Exports contain rows from `chunks`, in this exact order:

```text
chunk_id, page_path, title, type, heading_path, ordinal, sources, tags, updated, text
```

Exports do not include `score`, `mtime`, or FTS internals. Scores are
query-local and computed only by `search`.

## Ingest Use

1. Extract candidate entities, concepts, methods, datasets, and claims.
2. Search BM25 for existing pages before creating new pages.
3. Prefer updating an existing page when it represents the same durable idea.
4. Create a new page only when no suitable page exists.
5. After modifying `wiki/`, rebuild if `auto_rebuild_after_ingest: true`.

## Export

Use:

```bash
python3 scripts/wiki_fts.py export --format jsonl --out exports/bm25-chunks.jsonl
python3 scripts/wiki_fts.py export --format csv --out exports/bm25-chunks.csv
python3 scripts/wiki_fts.py export --format markdown --out exports/bm25-report.md
```

Exported files contain chunk rows with page metadata fields and chunk text, not
static BM25 scores. Scores are computed per query.
